# 新增功能使用指南

本文档介绍新增的两个前端功能：**文章版本历史查询** 和 **Token 刷新工具**。

## 1. 文章版本历史查询

### 功能说明

文章版本历史功能允许用户查看文章的所有历史版本，包括每次修改的内容、修改时间、修改人以及变更日志。

### API 位置

`src/api/graphql/blog.ts`

### 提供的 Hook

#### 1.1 `usePostVersions` - 自动加载版本历史

**使用场景：** 在组件加载时自动获取版本历史

```typescript
import { usePostVersions } from '@/api/graphql/blog';

function PostVersionHistory({ postId }: { postId: string }) {
  const { versions, loading, error, refetch } = usePostVersions(postId);

  if (loading) return <div>加载中...</div>;
  if (error) return <div>加载失败: {error.message}</div>;

  return (
    <div>
      <h2>版本历史</h2>
      <button onClick={() => refetch()}>刷新</button>
      {versions.map((version) => (
        <div key={version.id}>
          <h3>版本 {version.versionNum}</h3>
          <p>标题: {version.title}</p>
          <p>修改人: {version.createdBy.username}</p>
          <p>修改时间: {new Date(version.createdAt).toLocaleString()}</p>
          {version.changeLog && <p>变更说明: {version.changeLog}</p>}
        </div>
      ))}
    </div>
  );
}
```

**参数说明：**
- `postId: string` - 文章 ID
- `skip?: boolean` - 可选，是否跳过查询（默认 false）

**返回值：**
- `versions: BlogPostVersion[]` - 版本历史数组
- `loading: boolean` - 加载状态
- `error: ApolloError | undefined` - 错误信息
- `refetch: () => Promise<any>` - 重新获取数据

#### 1.2 `useLazyPostVersions` - 按需加载版本历史

**使用场景：** 点击按钮或触发某个事件后才加载版本历史

```typescript
import { useLazyPostVersions } from '@/api/graphql/blog';

function PostDetailPage({ postId }: { postId: string }) {
  const { fetchVersions, versions, loading, error } = useLazyPostVersions();
  const [showVersions, setShowVersions] = useState(false);

  const handleShowVersions = async () => {
    await fetchVersions(postId);
    setShowVersions(true);
  };

  return (
    <div>
      <h1>文章详情</h1>
      {/* 文章内容 */}

      <button onClick={handleShowVersions}>
        查看版本历史
      </button>

      {showVersions && (
        <div>
          {loading && <p>加载中...</p>}
          {error && <p>加载失败: {error.message}</p>}
          {versions.map((version) => (
            <div key={version.id}>
              <h3>版本 {version.versionNum}</h3>
              <p>{version.title}</p>
            </div>
          ))}
        </div>
      )}
    </div>
  );
}
```

**返回值：**
- `fetchVersions: (postId: string) => Promise<any>` - 手动触发查询
- `versions: BlogPostVersion[]` - 版本历史数组
- `loading: boolean` - 加载状态
- `error: ApolloError | undefined` - 错误信息

## 2. Token 刷新工具

### 功能说明

Token 刷新功能用于在 JWT Token 即将过期或已过期时，自动刷新获取新的 Token，避免用户频繁登录。

### API 位置

`src/api/graphql/auth.ts`

### 使用方法

#### 2.1 基础使用

```typescript
import { useAuth } from '@/api/graphql/auth';

function App() {
  const { refreshToken, loading, errors } = useAuth();

  const handleRefresh = async () => {
    try {
      const result = await refreshToken();
      console.log('Token 刷新成功:', result);
      // 新的 token 已自动保存到 localStorage
    } catch (error) {
      console.error('Token 刷新失败:', error);
      // 失败时会自动清除 token 并重定向到登录页
    }
  };

  return (
    <div>
      <button onClick={handleRefresh} disabled={loading.refresh}>
        {loading.refresh ? '刷新中...' : '刷新 Token'}
      </button>
      {errors.refresh && <p>错误: {errors.refresh.message}</p>}
    </div>
  );
}
```

#### 2.2 自动刷新 Token（推荐实现）

在应用根组件或 Apollo Client 配置中实现自动刷新：

```typescript
// src/hooks/useAutoRefreshToken.ts
import { useEffect, useRef } from 'react';
import { useAuth } from '@/api/graphql/auth';

export function useAutoRefreshToken() {
  const { refreshToken } = useAuth();
  const timerRef = useRef<NodeJS.Timeout>();

  useEffect(() => {
    // 解析 token 过期时间
    const token = localStorage.getItem('token');
    if (!token) return;

    try {
      const payload = JSON.parse(atob(token.split('.')[1]));
      const expiresAt = payload.exp * 1000; // 转换为毫秒
      const now = Date.now();
      const timeUntilExpiry = expiresAt - now;

      // 在过期前 5 分钟刷新 token
      const refreshTime = timeUntilExpiry - 5 * 60 * 1000;

      if (refreshTime > 0) {
        timerRef.current = setTimeout(async () => {
          try {
            await refreshToken();
            console.log('Token 自动刷新成功');
          } catch (error) {
            console.error('Token 自动刷新失败:', error);
          }
        }, refreshTime);
      }
    } catch (error) {
      console.error('解析 token 失败:', error);
    }

    return () => {
      if (timerRef.current) {
        clearTimeout(timerRef.current);
      }
    };
  }, [refreshToken]);
}

// 在 App.tsx 中使用
import { useAutoRefreshToken } from '@/hooks/useAutoRefreshToken';

function App() {
  useAutoRefreshToken(); // 启用自动刷新

  return <YourApp />;
}
```

#### 2.3 在 Apollo Error Link 中处理 Token 过期

```typescript
// src/graphql/client.ts 中添加
import { onError } from '@apollo/client/link/error';

const errorLink = onError(({ graphQLErrors, operation, forward }) => {
  if (graphQLErrors) {
    for (const error of graphQLErrors) {
      // 检测 token 过期错误
      if (error.extensions?.code === 'UNAUTHENTICATED' ||
          error.message.includes('token expired')) {

        // 尝试刷新 token
        return fromPromise(
          fetch('/graphql', {
            method: 'POST',
            headers: {
              'Content-Type': 'application/json',
            },
            body: JSON.stringify({
              query: `mutation { refreshToken { token refreshToken } }`
            })
          })
          .then(response => response.json())
          .then(({ data }) => {
            // 更新 localStorage
            localStorage.setItem('token', data.refreshToken.token);
            localStorage.setItem('refreshToken', data.refreshToken.refreshToken);

            // 重试原始请求
            const oldHeaders = operation.getContext().headers;
            operation.setContext({
              headers: {
                ...oldHeaders,
                authorization: `Bearer ${data.refreshToken.token}`,
              },
            });

            return forward(operation);
          })
          .catch(() => {
            // 刷新失败，重定向到登录页
            localStorage.removeItem('token');
            localStorage.removeItem('refreshToken');
            window.location.href = '/login';
          })
        ).flatMap(() => forward(operation));
      }
    }
  }
});
```

## 7. 常见问题

### Q1: 版本历史为空怎么办？

A: 确保：
1. 文章确实有修改历史
2. 后端 `postVersions` resolver 正常工作
3. 用户有权限查看该文章的版本历史

### Q2: Token 刷新失败怎么办？

A: 检查：
1. refreshToken 是否存在于 localStorage
2. refreshToken 是否已过期
3. 后端 refreshToken resolver 是否正常工作
4. 网络连接是否正常

### Q3: 如何判断 token 即将过期？

A: 解析 JWT token 的 `exp` 字段：

```typescript
function isTokenExpiringSoon(token: string, minutesBeforeExpiry: number = 5): boolean {
  try {
    const payload = JSON.parse(atob(token.split('.')[1]));
    const expiresAt = payload.exp * 1000;
    const now = Date.now();
    const timeUntilExpiry = expiresAt - now;
    return timeUntilExpiry < minutesBeforeExpiry * 60 * 1000;
  } catch {
    return true; // 解析失败时视为即将过期
  }
}
```
